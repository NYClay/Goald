# Data Model — Goald

> Firestore collection structure and document paths. Read before changing any service or type.

---

## Collections

### `bears/{bearId}`

User's companion bear with level, XP, cosmetics, and mood.

| Field | Type | Description |
|---|---|---|
| `userId` | `string` | Owner's Firebase Auth UID |
| `name` | `string` | Bear's display name |
| `level` | `number` | Current level (1-10) |
| `xp` | `number` | Cumulative experience points |
| `size` | `Size` | Derived from level: `cub`, `grown`, `giant` |
| `accessories` | `AccessoryId[]` | Unlocked cosmetic items |
| `mood` | `Mood` | Derived from feed recency: `hungry`, `content`, `happy`, `ecstatic` |
| `createdAt` | `Timestamp` | Creation timestamp |
| `updatedAt` | `Timestamp` | Last update timestamp |

**Document ID pattern:** `bear_{userId}` (one bear per user)

**Security rule:** Users can only read/write their own bear document.

---

### `caves/{caveId}`

A savings goal represented as a furnishable cave.

| Field | Type | Description |
|---|---|---|
| `userId` | `string` | Owner's Firebase Auth UID |
| `name` | `string` | Cave/goal display name |
| `targetAmount` | `number` | Savings target in cents |
| `currentAmount` | `number` | Current balance in cents |
| `furniture` | `FurnitureId[]` | Progressively unlocked furniture items |
| `theme` | `CaveTheme` | Visual theme: `cozy`, `modern`, `rustic`, `spa` |
| `createdAt` | `Timestamp` | Creation timestamp |
| `completedAt` | `Timestamp?` | Set when target is reached |

**Document ID:** Auto-generated by Firestore.

**Security rule:** Users can only read/write their own cave documents.

---

### `missions/{missionId}`

Daily mission for the user.

| Field | Type | Description |
|---|---|---|
| `userId` | `string` | Owner's Firebase Auth UID |
| `type` | `MissionType` | `save_amount`, `round_up`, `skip_purchase`, `custom` |
| `target` | `number` | Target value (amount in cents or count) |
| `completed` | `boolean` | Whether the mission is done |
| `xpReward` | `number` | XP awarded on completion |
| `date` | `string` | ISO date string (`YYYY-MM-DD`) |
| `completedAt` | `string?` | ISO timestamp when completed |

**Document ID:** Auto-generated by Firestore.

**Security rule:** Users can only read/write their own mission documents.

---

### `streaks/{streakId}`

Mission completion streak tracking.

| Field | Type | Description |
|---|---|---|
| `userId` | `string` | Owner's Firebase Auth UID |
| `current` | `number` | Current consecutive-day streak |
| `longest` | `number` | All-time longest streak |
| `lastMissionDate` | `string` | ISO date of last completed mission |
| `fireLevel` | `0 \| 1 \| 2 \| 3` | Visual fire animation level |

**Document ID pattern:** `streak_{userId}` (one streak per user)

**Security rule:** Users can only read/write their own streak document.

---

## References

All document references use the Firestore document reference pattern:
- Bear → `bears/{bearId}`
- Cave → `caves/{caveId}`
- Mission → `missions/{missionId}`
- Streak → `streaks/{streakId}`

There are no cross-document joins. Each document is self-contained with a `userId` field for security rule enforcement.

### `notifications` (placeholder)

`firestore.rules` defines a `notifications/{notificationId}` collection with owner-scoped CRUD rules. **No service, hook, or UI reads or writes to this collection.** The rules exist as scaffolding for a future push notification system (FCM). See `KNOWN_LIMITATIONS.md` §9.

## Derived Data

Some values are derived at read time, not stored:

| Value | Derived From | Function |
|---|---|---|
| Bear `size` | `level` | `calculateSize()` in `src/utils/bearUtils.ts` |
| Bear `mood` | `lastFeedDate`, `xp` | `calculateMood()` in `src/utils/bearUtils.ts` |
| Cave progress | `currentAmount / targetAmount` | `getCaveProgress()` in `src/utils/bearUtils.ts` |
| Earned badges | `stats`, `goals` | `computeEarnedBadges()` in `src/utils/badges.ts` |
| Milestone crossings | `prevProgress`, `nextProgress` | `getCrossedMilestones()` in `src/utils/depositUtils.ts` |
